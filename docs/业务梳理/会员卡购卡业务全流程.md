以下是一个标准的高频线上购卡流程（以小程序/App购买年卡、月卡为例）：

### 完整流程示例（健身房会员卡购买）

#### 1. 选择商品（卡种）

- **用户**：浏览会员卡类型（月卡/季卡/年卡/次卡）、价格、权益（是否含团课/泳池）、有效期规则（购卡后X天内未开卡可退，开卡后计时）→ 点击“立即购买”
- **系统**：校验该用户是否已有有效会员卡（防重复购卡）、展示适用门店

#### 2. 下单（生成权益订单）

- **用户**：确认购卡人信息（姓名+手机号）、选择开卡门店(根据当前项目业务而定，如果还是单体单店铺，就不用，后续再拓展加入)、勾选《会员协议》→ 点击“提交订单”
- **系统**：
  - 生成订单号，状态 = **待支付**
  - 记录开卡规则：例如“付款后需在30天核销激活，否则自动退款/失效”
  - 设定支付倒计时（后续使用延迟队列实现）

#### 3. 支付

- **用户**：微信/支付宝完成支付（通常无需银行卡）
- **系统**：
  - 支付成功后，订单状态 = **已支付/未激活**
  - 生成**电子卡券码**（或二维码）→ 发送到用户“我的卡包”
  - 短信通知：“请到店出示二维码开卡”

#### 4. 核销激活（关键步骤，替代实物发货）

- **用户**：携带手机到健身房前台 → 出示二维码/卡券码
- **工作人员**：扫码枪/App核销 → 系统验证是否未过期、未使用
- **系统**：
  - 核销成功 → 订单状态变为 **已激活**
  - 开始计时（年卡：365天；月卡：30天）
  - 该会员身份同步到闸机系统（刷脸/刷卡入场）

#### 5. 入场使用与后续流程

- **用户**：后续每次通过人脸识别或手机扫码进店锻炼
- **系统**：记录入场时间、剩余天数（会员端可见）、签到记录

#### 6. 售后服务（特有）

- **停卡/延期**：如因伤病可申请暂停会员卡（冻结天数）
- **转卡**：部分健身房允许付费转给他人
- **退款**：未激活状态下通常可退；激活后不可退（按合同）

***

### 与实物购买的核心差异总结

| 环节       | 实物电商            | 健身房会员卡              |
| -------- | --------------- | ------------------- |
| **支付后**  | 等待发货            | 等待到店核销激活            |
| **履约方式** | 物流配送            | 人工扫码核销 + 权益计时       |
| **核心状态** | 待发货 → 已发货 → 已收货 | 未激活 → 已激活（计时中）→ 已过期 |
| **超时处理** | 超时未支付取消订单       | 超时未核销可自动退款/失效       |
| **长期动作** | 售后（7天无理由）       | 入场记录、暂停、转卡、续费提醒     |
| **关联硬件** | 无               | 闸机、人脸识别设备、前台核销PDA   |

***

### 一个常见变体：纯线上购卡 + 自助开卡（当前项目推荐的方案）

**许多健身房也支持“付款后立即线上激活”**（无需到店核销）：

- 支付成功后，用户自行点击“立即开卡”按钮 → 计时开始 → 会员二维码生效 → 下次到店直接刷码入场

**此时流程变为**：\
选择卡种 → 下单 → 支付 → **用户主动点击开卡** → 权益生效（门店可跳过核销步骤）
