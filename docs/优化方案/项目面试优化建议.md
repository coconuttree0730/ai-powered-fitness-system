## 二、当前已有亮点总结（面试可以讲的）

### ⭐⭐⭐ 亮点一：自研Agent架构（最高含金量）

你的 [JianXiaoZhuAgentServiceImpl](file:///g:/ai-powered-fitness-system-v2/src/main/java/com/fitness/modules/chat/agent/JianXiaoZhuAgentServiceImpl.java) 基于 Spring AI Alibaba 的 **ReactAgent**，实现了：

```java
ReactAgent.builder()
    .name("jian_xiao_zhu_agent")
    .model(chatModel)
    .methodTools(membershipQueryTools, courseQueryTools, productQueryTools, 
                 coachQueryTools, profileQueryTools, dateTimeQueryTools,
                 weatherQueryTools, locationQueryTools, ragQueryTool)
    .hooks(summarizationHook)     // 对话摘要Hook
    .saver(redisSaver)            // Agent状态持久化
    .build();
```

- 9个业务Tool可被LLM自动调度调用
- SSE流式输出区分 delta / tool_start / tool_finish / model_finished 四种事件
- 用 RedisSaver 持久化 Agent Graph 的 checkpoint，实现**断点续聊**
- SummarizationHook 防止长对话上下文超限

这是**校招面试中极其少见的Agent实践**，远超市面上"调API"的AI项目。

### ⭐⭐⭐ 亮点二：混合检索 RAG（信息检索专业度）

你的 [RAGServiceImpl](file:///g:/ai-powered-fitness-system-v2/src/main/java/com/fitness/modules/knowledge/service/impl/RAGServiceImpl.java) 实现了：

- **双路召回**：向量检索（Ollama Embedding）+ 关键词检索（数据库全文搜索）
- **RRF融合排序**（Reciprocal Rank Fusion）— 这是搜索引擎领域的经典算法
- 分类过滤 + 相似度阈值控制

面试时对比"只会调Embedding API"的竞争者，你可以讲清楚 **检索的Precision/Recall权衡**、**混合检索比纯向量检索的优势**。

### ⭐⭐ 亮点三：数据库版本管理

49个Flyway迁移脚本，展示了从V1到V49的完整数据库演进过程。校招项目中几乎没有这种工程化实践。

### ⭐⭐ 亮点四：分层异常处理体系

15+种异常类型的精细化处理，包括ClientAbortException（客户端断开）的处理——这在生产环境经常遇到。

### P1 — 增强含金量（1-2周，面试重点加分项）

#### 5. 分布式事务（Seata AT模式 / TCC）

当前项目有 **商品购买 → 扣库存 → 积分扣减 → 支付** 这条链路，每个操作在不同模块，存在分布式事务问题。

```java
@GlobalTransactional  // Seata AT模式
public OrderResult purchaseProduct(Long userId, Long productId) {
    productService.deductStock(productId);    // 商品模块
    pointsService.deductPoints(userId, price); // 用户模块
    paymentService.pay(userId, productId);     // 支付模块
    orderService.createOrder(...);             // 订单模块
}
```

**面试价值**：能讲清楚2PC/TCC/Saga的区别和Seata的实现原理，属于高级话题。

#### 6. 消息队列异步化（RabbitMQ / RocketMQ）

将以下场景改为异步：

```
同步 → 异步改造：
├── 短信发送         → MQ异步（高延迟IO不阻塞主线程）
├── AI健身计划生成    → MQ异步 + WebSocket推送结果
├── 数据分析报告生成   → MQ异步 + 轮询或SSE通知
└── 订单超时取消      → 延迟队列（RocketMQ延迟消息）
```

**面试价值**：能讲清"削峰填谷"、"最终一致性"、"死信队列"等概念。

#### 7. 服务熔断降级（Sentinel / Resilience4j）

AI服务、短信服务、支付宝都是外部依赖，需要熔断保护。

```java
@CircuitBreaker(name = "aiService", fallbackMethod = "aiFallback")
public FitnessPlanResponseDTO generateFitnessPlan(Map<String, Object> profile) {
    return aiService.generateFitnessPlanFromProfile(profile);
}

// 降级策略：AI不可用时使用规则引擎生成模板计划
public FitnessPlanResponseDTO aiFallback(Map<String, Object> profile, Throwable t) {
    return ruleEnginePlanGenerator.generate(profile);
}
```

**面试价值**：能讲清熔断三种状态（Closed/Open/Half-Open）和滑动窗口统计。

#### 8. 读写分离 + 数据库连接池优化

```yaml
# 配置主从数据库
spring:
  datasource:
    master:  # 写库
      hikari:
```

自定义 `RoutingDataSource` + `@ReadOnly` 注解实现无侵入的读写分离。

用自定义 `@ApiVersion` 注解 + RequestMappingHandlerMapping 拦截实现。

## 五、面试话术框架

### 项目介绍（30秒版本）

> "我做了一个智能健身房管理系统，后端基于Spring Boot 3.5 + Spring AI Alibaba。核心亮点是**自研了AI Agent架构**——用ReactAgent集成了9个业务工具函数，让LLM能自动调用课程查询、会员管理、健身计划生成等接口。还实现了**混合检索RAG**——把用户上传的健身知识文档向量化，用向量+关键词双路召回+RRF融合排序，提升检索精准度。整个项目有49套Flyway数据库迁移、31个单元测试，准备容器化后用Docker Compose一键部署。"

### 被追问"Agent怎么做的"时

> "我用Spring AI Alibaba的ReactAgent框架，核心是Think-Act-Observe循环。配置了RedisSaver做Agent状态的checkpoint持久化，这样会话中断后可以恢复。加了SummarizationHook做自动摘要，当历史消息超过1000 token时触发摘要压缩。工具调用这层，LLM通过Function Calling可以调用9个业务工具，包括查课程、查会员卡、查商品、查教练等。流式输出上我区分了4种事件——delta增量文本、tool_start工具开始调用、tool_finish工具调用完成、model_finished模型输出完成——前端可以根据不同事件做不同渲染。"

### 被追问"RAG怎么做的"时

> "我用的是混合检索方案。入库时用Ollama的Embedding模型把文档chunk做向量化存到pgvector。查询时双路并行——一路向量相似度检索，一路PostgreSQL全文关键词搜索，各取topK的两倍做候选集。融合这步用的是RRF算法（Reciprocal Rank Fusion），给两路结果按排名计算加权分数，再做rerank。最后把topK结果拼接成上下文喂给LLM生成答案。对比纯向量检索，混合检索在专业术语和数字查询场景下召回率明显更好。"
