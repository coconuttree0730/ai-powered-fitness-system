## 🎯 拆分目的

将原有的单一知识库文档拆分为6个独立文档，以提升RAG检索的准确性和效率：

- ✅ **减少语义漂移**：避免"停卡规则"匹配到"停车规则"
- ✅ **降低检索噪声**：用户问课程时不会返回器械使用规则
- ✅ **提升响应速度**：减少无关向量计算
- ✅ **提高回答质量**：LLM接收更精准的上下文

## 🔧 技术实现

### RAG检索流程

```
用户提问
    ↓
查询路由器（可选）
    ↓
分类识别 → categoryId
    ↓
向量检索 + 关键词检索
    ↓
RRF融合排序
    ↓
Top-K结果
    ↓
构建上下文
    ↓
LLM生成回答
```

### 查询路由器示例代码

```java
@Service
public class QueryRouterService {
    
    public Long routeQuery(String query) {
        // 基于关键词的分类路由
        if (query.matches(".*价格|多少钱|费用|收费.*")) {
            return 2L; // 会员卡与收费政策
        }
        if (query.matches(".*课程|约课|团课|私教.*")) {
            return 3L; // 课程与预约服务
        }
        if (query.matches(".*教练|张磊|李娜|王强|刘芳.*")) {
            return 6L; // 教练团队介绍
        }
        if (query.matches(".*怎么|如何|操作|流程.*")) {
            return 5L; // FAQ
        }
        // 默认全库搜索
        return null;
    }
}
```

## ❓ 常见问题

### Q1：为什么要拆分而不是保持一个文档？

**A**：拆分后的优势：
1. **语义更精准**：每个分类的主题更聚焦，向量表示更准确
2. **检索更快**：可以只搜索相关分类，减少计算量
3. **维护更方便**：更新某个分类的内容不会影响其他分类

### Q2：分类ID会自动分配吗？

**A**：是的，执行SQL后，PostgreSQL会自动分配ID（通常从1开始）。上传文档时需要根据实际ID对应。

### Q3：如何验证拆分效果？

**A**：对比测试：
1. 使用相同问题分别测试拆分前后的回答
2. 检查返回的文档来源是否更精准
3. 统计用户满意度变化

### Q4：可以自定义分类吗？

**A**：当然可以！你可以：
1. 修改SQL中的分类名称和描述
2. 调整文档内容归属
3. 增加新的分类（如"营养饮食指南"）
