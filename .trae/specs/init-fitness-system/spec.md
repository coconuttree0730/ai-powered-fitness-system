# 智能健身房管理系统 MVP 规格说明书

## 为什么

构建一个基于 Spring Boot 的智能健身房管理系统，为会员、教练、管理员(未登录的游客可浏览网站官网首页)提供一体化的健身服务与运营管理平台。MVP 版本聚焦于核心功能实现，为后续高级功能（RAG、支付、消息队列等）奠定基础。

## 变更内容

- **新增**: 用户认证授权系统（Spring Security + JWT）
- **新增**: 多角色支持（学员、教练、管理员、游客）
- **新增**: 课程管理与预约系统
- **新增**: AI 智能健身计划生成功能（Spring AI Alibaba）
- **新增**: 器材管理与报修系统
- **新增**: 运营数据可视化分析（ECharts + AI 分析）
- **新增**: MinIO 图片存储集成

## 影响范围

- 新增模块：user、course、equipment、plan、dashboard
- 数据库表：用户表、角色表、课程表、预约表、器材表、维护日志表、健身计划表
- 外部依赖：阿里百炼平台 LLM、MinIO 对象存储

## 新增需求

### 需求 1: 用户认证与权限管理

**描述**: 实现学员、教练、管理员三类角色的注册、登录、密码修改功能，支持游客模式访问官网页面（店铺介绍，教练员介绍，器械介绍，课程介绍,...）。

#### 场景 1.1: 用户注册

- **GIVEN** 用户访问注册页面
- **WHEN** 提交手机号、密码 或者 手机号 、短信验证码
- **THEN** 系统创建用户账号，密码使用 BCrypt 加密存储

#### 场景 1.2: 用户登录

- **GIVEN** 用户已注册账号
- **WHEN** 提交正确的手机号和密码 或者 手机号 、短信验证码
- **THEN** 系统返回 JWT Token，Token 同时在服务端记录（用于后续校验和登出）

#### 场景 1.3: 游客访问

- **GIVEN** 用户未登录
- **WHEN** 访问官网首页、课程轮播图、公开课程列表、教练员展示列表
- **THEN** 允许访问，无需认证

#### 场景 1.4: 权限控制

- **GIVEN** 用户已登录
- **WHEN** 访问需要特定角色的资源
- **THEN** 系统根据用户角色进行权限校验

### 需求 2: 课程管理与预约

**描述**: 管理员发布健身公开课课程、配置排期；学员查看课程并完成预约。

#### 场景 2.1: 管理员发布课程

- **GIVEN** 管理员已登录
- **WHEN** 填写课程信息（名称、描述、教练、时间、容量、分类）
- **THEN** 课程发布成功，学员可见

#### 场景 2.2: 学员预约课程

- **GIVEN** 学员已登录，课程有余量
- **WHEN** 点击预约按钮
- **THEN** 预约成功，记录预约信息

#### 场景 2.3: 学员取消预约

- **GIVEN** 学员已预约课程
- **WHEN** 在课程开始前取消预约
- **THEN** 取消成功，释放课程名额

#### 场景 2.4: 课程列表展示

- **GIVEN** 用户访问课程页面
- **WHEN** 加载页面
- **THEN** 展示课程列表，支持按分类筛选，游客也可查看

### 需求 3: AI 智能健身计划生成

**描述**: 学员身体数据（个人主页个人信息填写获取）、健身目标、需要训练部位；系统调用 LLM 生成个性化周度（7天）健身计划，生成精美的前端效果展示。

#### 场景 3.1: 完善个人信息

- **GIVEN** 学员已登录
- **WHEN** 在"完善信息"菜单填写身高、体重、年龄、健身经验等
- **THEN** 信息保存成功

#### 场景 3.2: 生成健身计划

- **GIVEN** 学员已完善个人信息
- **WHEN** 输入健身目标（增肌/减脂/塑形）、训练部位偏好
- **THEN** 系统调用阿里百炼 LLM，生成个性化周度健身计划

#### 场景 3.3: 查看健身计划

- **GIVEN** 学员已生成健身计划
- **WHEN** 访问健身计划页面
- **THEN** 展示生成的周度计划详情

### 需求 4: 器材管理

**描述**: 管理员维护器材信息、查看维护记录；学员查看器材、上报故障。

#### 场景 4.1: 管理员器材 CRUD

- **GIVEN** 管理员已登录
- **WHEN** 添加/编辑/删除/查询器材信息
- **THEN** 器材数据正确维护

#### 场景 4.2: 学员查看器材

- **GIVEN** 学员已登录
- **WHEN** 访问器材列表或详情
- **THEN** 展示器材信息（名称、位置、状态、使用说明）

#### 场景 4.3: 学员报修器材

- **GIVEN** 学员发现器材故障
- **WHEN** 上传故障图片、填写故障描述并提交
- **THEN** 报修记录保存，图片存储到 MinIO

#### 场景 4.4: 管理员查看维护记录

- **GIVEN** 管理员已登录
- **WHEN** 访问维护日志页面
- **THEN** 展示所有报修记录和维修记录

### 需求 5: 运营数据可视化分析

**描述**: 管理员通过 ECharts 查看运营数据，并使用 AI 进行智能分析。

#### 场景 5.1: 数据可视化展示

- **GIVEN** 管理员已登录
- **WHEN** 访问数据看板
- **THEN** 展示会员卡销量（月卡/季卡/年卡）、到店高峰时间等图表

#### 场景 5.2: AI 智能分析

- **GIVEN** 管理员在数据看板页面
- **WHEN** 点击"智能分析"按钮
- **THEN** 系统调用 LLM 分析当前运营数据，返回分析报告

## 技术架构

### 后端技术栈

- Spring Boot 3.5.x
- Spring Security + JWT
- Spring AI Alibaba 1.1.2.0
- MyBatis-Plus 3.5.x
- PostgreSQL 16（业务数据 + 向量存储，用于RAG回答）
- MinIO（图片/视频存储）
- Flyway 10.x（数据库版本控制）

### 前端技术栈

- Vue 3
- Vue Router
- Pinia
- Axios
- ECharts
- Element Plus（后台管理端）
- Naive UI（前台会员端、教练员端、）

## 数据库设计概要

### 核心表

1. `sys_user` - 用户表
2. `sys_role` - 角色表
3. `sys_user_role` - 用户角色关联表
4. `fitness_course` - 课程表
5. `fitness_booking` - 预约表
6. `fitness_equipment` - 器材表
7. `fitness_equipment_repair` - 器材报修表
8. `fitness_plan` - 健身计划表
9. `fitness_plan_detail` - 健身计划详情表
10. `sys_file` - 文件记录表（MinIO）
11. `sys_permission`\*\* \*\*- **权限表**
12. `sys_role_permission`\*\* **- \*\* 角色权限关联表**

## API 设计规范

- 基础路径: `/api/v1/资源名`
- 响应格式: 统一使用 `Result<T>` 包裹
- 认证方式: JWT Token（Header: Authorization: Bearer {token}）
- 权限控制: 基于角色的访问控制（RBAC）
